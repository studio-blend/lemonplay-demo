🔴 CRITICAL — All PagesMeta description is a developer placeholder on every single page
Every page on the site — homepage, about, contact, legal, careers — has an identical meta description: "Next.js app with scroll-based sections". This is a development template placeholder that was never replaced. Google displays this in SERPs, destroying click-through rates and signalling an unprofessional, incomplete site. No page has a specific title. About, Contact, Solutions, Careers — all render the same <title>Lemonpay</title>. This eliminates all on-page SEO, makes browser tab management impossible, and prevents Google from understanding page context.

Verdict

The design team behind this site appears to have been a solo developer or small team using a drag-and-drop builder (Builder.io) without a dedicated designer. The dark colour palette and yellow accent show awareness of fintech visual conventions, but the execution lacks a systematic design language. Placeholder assets were shipped to production (TEMP CDN logos, empty photo slots, developer meta tags), indicating the site went live before it was ready. To the primary audience — SME founders evaluating payment partners — the site signals early prototype rather than trusted payment infrastructure. The gap between Lemonpay's claimed capabilities (AI-KYC, 500+ merchants, 96% satisfaction) and the site's visual maturity creates cognitive dissonance that directly reduces merchant conversion rates.
